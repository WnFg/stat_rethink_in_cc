## 一、画像 Snapshot（2026-06-20 起）
- **背景**：数学（微积分/线代）生疏；Python 编程底子可以，但 NumPy/SciPy/PyMC 等库用得少；统计上频率派+贝叶斯都接触过、有基础；目标是**工作/数据分析**提升。
- **强项**：直觉强、迁移快；**爱自己推导**而非套公式（自推过数值二阶导）；能一眼看穿本质（发现 `binom.pmf(w,1,grid)` 就是 p/1−p）。
- **弱项**：① 频率派直觉带来的解读混淆（可信区间≠置信区间）；② 易把"先验权重 prior(p)"与"参数值 p"混；③ 偶发运算符优先级等代码细节。
- **兴趣点**：**刨根问底数学原理**（追问二次近似理论基础到 σ²=1/|f''|）；**API 机制**（主动问 binom.pmf/minimize/np.clip 在做什么）；频率派↔贝叶斯的桥（MLE/MAP）。

## 二、教学风格参数 Style knobs（skill 读取并应用）
| 维度 | 取值 | 落地做法 |
|------|------|---------|
| 数学深度 | **中高**（爱深究） | 先直觉后数学；推导要给，但讲清每步动机；可顺着他的追问延伸理论 |
| 练习形式 | **OJ 优先** | 定函数签名+隐藏测试，学生实现、`tools/judge.py` 判定；少给现成答案 |
| 反馈风格 | **鼓励自推** | 先让他说思路再写码；卡住给提示不给答案；肯定他的自推/洞察 |
| 可视化 | **浏览器看图** | 关键机制画 PNG 存 `materials/figures/`，`open -a Safari` 给他看；**不用点阵/ASCII 图** |
| 公式呈现 | **纯文本/HTML** | 讲解里**不用 LaTeX**（终端不渲染）；复习卡用 HTML |
| 例子取向 | **业务化** | 多用转化率/AB 测试/计数分类，少用纯学术题 |
| 语言 | 中文为主+英文术语 | posterior/likelihood 等保留英文 |
| 节奏 | 每节 20–30 分钟、按小节 | — |
| 代码分工 | **口述算法→老师写实现** | 学生主导算法设计与 OJ 自写；口述完算法/步骤后由老师完成 API 翻译；老师不应因此自我批评 |
