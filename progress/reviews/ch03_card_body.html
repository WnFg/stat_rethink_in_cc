② 数学思想与作者核心观点

McElreath 在本章反复强调的"为什么"，比算法更重要。每条可追溯原文。

随机性是信息的属性，不是真实世界的属性 — 后验里的"随机抽样"是数学装置：参数在计算机里和抛硬币一样"经验"， 但现实世界的参数本身不随机。"随机性"只描述我们对它的无知。 §3.1, p63
采样把积分变成数据汇总 — 区间 = 积分，对多参数模型极难。换成样本后，区间 = 数样本比例， 哪怕一个非数学家也能回答复杂的后验问题。这是本章的核心工程动机。 §3.1, p64
提早学样本是为 MCMC 做准备 — 最强大的后验计算方法（MCMC，第 8 章）只给样本，不给公式。 现在学会处理样本，以后水到渠成。 §3.1, p64
整条后验才是贝叶斯的估计 — 区间、点估计都只是摘要，不是估计本身。 "若换个区间类型结论就变，那就别用区间，直接画整条后验。" §3.2, p70
可信区间 ≠ 置信区间；95% 不意味着"95% 的时候含真值" — 贝叶斯可信区间：P(p ∈ 区间 | 数据) = 0.95，概率属于参数。 频率派置信区间：重复实验 95% 的区间盖住真值，概率属于程序。 两者都不能说"95% 的时候含真值"——置信区间存在长期慢性过度自信（chronic overconfidence）。 §3.2, p71
模型检查不是判断对错，而是看怎么失败 — "所有模型都是错的"。PPC 的目标是理解模型在哪个角度失败、失败得要不要紧。 Golem 想象力很差——你必须主动换角度（游程、切换次数…），检验才有力度。 §3.3, p78–81

关键数学思想

  • 采样的数学本质：rng.choice(grid, p=posterior) 让每个 grid 值按 posterior 权重出现，大样本时频率 → 密度。这是蒙特卡洛积分的最简形式：E[f(p)] ≈ (1/N)Σf(sample_i)。
  • HPDI 的最优性：在所有含指定质量 q 的区间中，HPDI 是使区间宽度最小的那个。偏态时它总包含后验众数（MAP）。
  • 后验预测 = 两层随机：先从后验抽 p（参数不确定），再从 Binomial(n,p) 抽观测（观测不确定）。合并两层才是诚实预测；只用点估计会低估预测宽度。
  • Loss function 决定点估计：绝对损失 |d-p| 最小化 → 中位数；平方损失 (d-p)² → 均值。点估计的选择是决策问题，不是统计问题。

③ 核心概念与公式

natural frequencies 自然频率 — 把概率换成计数更直觉（"1094 阳性里 95 真吸血鬼"）。采样正是把后验变成频率格式。§3.1, p62–63
sampling 采样 — rng.choice(grid, size=N, p=posterior)：每个 p 按后验权重出现，频率 ≈ 密度。
PI 百分位区间 — np.percentile(s, [lo%, hi%])，两尾等切；偏态时漏掉最可信值。§3.2, p69
HPDI 最高后验密度区间 — 含指定质量的最窄区间；滑动窗口：排序 → 枚举 x[i+w]-x[i] → 取最小。§3.2, p69–70
posterior predictive distribution 后验预测分布 — rng.binomial(n, samples)：对每个后验 p 模拟一次，合并两种不确定性。§3.3, p78–79
PPC 后验预测检查 — 换角度看数据（游程、切换次数…）；观测落分布尾部 = 模型该角度失败。§3.3, p77–81
P(a ≤ p < b) = np.mean((samples >= a) & (samples < b))
后验预测：ppc = rng.binomial(n, samples)　← 一行，含两层不确定性

posterior = binom.pmf(W, N, p_grid); posterior /= posterior.sum()

# §3.1 采样代积分
P_ab = np.mean((samples >= a) & (samples < b))

# §3.2 PI（等尾）
PI90 = np.percentile(samples, [5, 95])

# §3.2 HPDI（最窄，滑动窗口）
def hpdi(samples, prob=0.9):
    x = np.sort(samples); n = len(x); w = int(np.floor(prob * n))
    widths = x[w:] - x[:n-w]; i = np.argmin(widths)
    return x[i], x[i+w]

# §3.3 后验预测（诚实版，含参数不确定性）
