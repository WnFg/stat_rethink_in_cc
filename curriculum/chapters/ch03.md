# 第 3 章 Sampling the Imaginary — 教案

> **出处**：McElreath《Statistical Rethinking》(1st ed) 第 3 章，PDF p62–83（书页 49–70）。Python 为教材 R 代码的等价翻译，页码均指 `tools/pdftext.py` 的 0 基 PDF 页。本章不需要 PyMC，NumPy/SciPy 足够。

**本章定位**：后验**算出来**只是开始，**用好它**才是本事。本章给你一把贯穿全书的万能钥匙——**从后验中采样**：把"积分问题"变成"数样本问题"（frequency format），再用样本去汇总（区间/点估计）、模拟预测、检查模型。
**关键开篇（p62–65）**：吸血鬼血检例子——检测准（95%）、假阳率低（1%），但吸血鬼稀有（0.1%），于是阳性里**真吸血鬼只占 8.7%**。换成"自然频率"（10 万人里 100 吸血鬼…95+999 阳性）一下就懂了。同构于 HIV/DNA 检测、甚至**显著性检验**（p64–65：基率极低——100 个假设里只 1 个为真——时，即便检验 power 95%、假阳率 5%，阳性结果也只有约 **16%** 概率对应真假设；把假阳率压到 1% 也才到约 **0.5（掷硬币水平）**）——**低基率下，再准的检验也救不了坏科学，关键是提高基率，要靠想而非靠测**。
**里程碑**：能从后验采样，算 PI/HPDI 与点估计，并做一次后验预测检查（PPC）。
**数据**：globe tossing（6/9、3/3、8/15）；Hard 用出生性别数据 `birth1/birth2`（书 p82，R code 3.28；上课时放到 `materials/data/`）。
**工具**：NumPy（`np.random.choice`、`np.random.binomial`、`np.percentile`）、SciPy。
**上课规划（约 4 节）**：① 3.1 采样入门 → ② 3.2 区间(PI/HPDI) → ③ 3.2 点估计+loss / 3.3 模拟与后验预测 → ④ `/review` 章末复习。

## 3.1 Sampling from a grid-approximate posterior（PDF p62–66）

- **学习目标**：学会从网格后验中抽样，并用"数样本"代替"算积分"回答关于后验的问题。
- **核心概念**：
  - **自然频率 natural frequencies**（p62–63）：把概率换成计数（"1094 个阳性里 95 个真吸血鬼"）更符合直觉。本章正是把后验当概率分布、**从中采样产生计数**。
  - **为什么早学采样**（p64）：① 把微积分（区间=积分）变成**数据汇总**（区间=数样本）；② 最强的后验计算法（MCMC，第 8 章）**只产样本**，先学会处理样本，以后水到渠成。
  - **采样**（§3.1, p65）：`samples = np.random.choice(p_grid, size=N, p=posterior)`——把后验想成一个装着各 p 值的"桶"，每个值按其后验概率出现，舀 N 个出来，频率≈后验密度。
- **关键直觉**：后验是"可信度"的分布；从中采样，得到的样本里**每个 p 出现的频率正比于它的后验可信度**（p65）。注意（p63）：这里的"随机"是数学装置，**随机性是信息的属性，不是真实世界的属性**——计算机里参数和抛硬币一样"经验"。
- **Worked example**（书 R code 3.2–3.7 的 Python 等价）：
  ```python
  import numpy as np
  from scipy.stats import binom
  p_grid = np.linspace(0, 1, 1000)
  prior = np.ones(1000)
  likelihood = binom.pmf(6, 9, p_grid)
  posterior = likelihood * prior
  posterior = posterior / posterior.sum()
  rng = np.random.default_rng(100)
  samples = rng.choice(p_grid, size=10000, p=posterior)
  # 用样本回答"p<0.5 的后验概率"，对比网格直接求和
  by_grid    = posterior[p_grid < 0.5].sum()
  by_samples = np.mean(samples < 0.5)
  print(round(by_grid, 3), round(by_samples, 3))   # ≈0.172 与 ≈0.17（采样有微小波动）
  ```
- **设计练习（OJ）**：实现 `prob_between(samples, lo, hi)` —— 用样本估计后验落在 `[lo, hi)` 的概率（频率法）。
  <details><summary>参考解与隐藏测试</summary>

  ```python
  # solution+tests: ch3 ex 3.1A
  import numpy as np
  from scipy.stats import binom
  def prob_between(samples, lo, hi):
      return np.mean((samples >= lo) & (samples < hi))
  # 构造 globe 6/9 后验样本
  p = np.linspace(0, 1, 1000); post = binom.pmf(6, 9, p); post /= post.sum()
  s = np.random.default_rng(100).choice(p, size=20000, p=post)
  assert abs(prob_between(s, 0.0, 0.5) - post[p < 0.5].sum()) < 0.02     # 与网格一致
  assert abs(prob_between(s, 0.5, 0.75) - 0.606) < 0.03                  # 书 R code 3.8 ≈0.61
  assert np.isclose(prob_between(s, 0.0, 1.01), 1.0)                     # 全区间=1
  print("3.1A ok")
  ```
  </details>
- **关联书题**：3E1–3E3（p82，用 `set.seed(100)` 的样本算 p<0.2 / p>0.8 / 0.2–0.8 之间的后验概率）。
- **常见误区**：
  - ❌ "采样是某种物理随机过程" → 是数学装置；随机性是信息属性（p63）。
  - ❌ "样本结果和网格必须完全相等" → 采样有波动，量大才趋近（p66）。

## 3.2 Sampling to summarize（PDF p66–73）

- **学习目标**：用样本做两类汇总——**区间**（PI vs HPDI）与**点估计**（靠 loss function 选）。
- **核心概念**：
  - **两类区间**（p66–69）：① **定边界**（boundaries）问"p<0.5 的概率多少"——`np.mean(samples<0.5)`；② **定质量**（mass）问"含 80% 质量的区间在哪"——分位数 `np.percentile`。
  - **PI 百分位区间**（percentile interval, p69）：两尾各留等量质量。**直观、好沟通，但偏态时会漏掉最可信值**。
  - **HPDI 最高后验密度区间**（highest posterior density interval, p69–70）：**含指定质量的最窄区间**，**总是包含峰**。偏态后验时与 PI 差很大（3 水/3 抛例子：PI50% ≈[0.70,0.93]、**宽 0.23**，漏掉 p=1；HPDI50% ≈[0.84,1.0]、**只宽 0.16**，含峰更窄，p70）。代价：计算贵、对样本量敏感、受众不熟。
  - **点估计**（p71–73）：MAP/mean/median 三者在偏态时不同。**选哪个取决于 loss function**：绝对损失 `|d−p|` → **median**；平方损失 `(d−p)²` → **mean**（p73）。
- **关键直觉**：
  - **整条后验才是贝叶斯的估计**（p70–71）；区间只是为了"描述形状"。**若换个区间类型结论就变，那就别用区间，直接画整条后验**（p70）。
  - **可信区间 ≠ 置信区间**（Rethinking, p71）：贝叶斯可信区间说"给定数据，p 在区间内的概率=95%"，写成 **`P(p ∈ 区间 | 数据) = 0.95`**（概率属于**参数**）；频率派置信区间说"重复实验 95% 的区间盖住真值"（概率属于**程序**）。
    书里还有更狠的一句（务必记住）：**"a 95% interval does not contain the true value 95% of the time"** —— 不管你用哪种解读，95% 区间都**不是**"95% 的时候含真值"。`true`（真值）这个词本身就该拉响警报：95% 只是个**小世界数字**，置信区间长期**慢性过度自信**（chronic overconfidence）。
- **Worked example A — PI vs HPDI（书 R code 3.11–3.13，3 水/3 抛偏态后验）**：
  ```python
  import numpy as np
  from scipy.stats import binom
  p = np.linspace(0, 1, 1000); post = binom.pmf(3, 3, p); post /= post.sum()
  s = np.random.default_rng(100).choice(p, size=20000, p=post)

  def hpdi(samples, prob=0.5):           # 最窄的含 prob 质量的区间
      x = np.sort(samples); n = len(x); w = int(np.floor(prob * n))
      widths = x[w:] - x[:n - w]; i = int(np.argmin(widths))
      return x[i], x[i + w]

  PI50 = np.percentile(s, [25, 75])      # 百分位区间（等尾）
  HPDI50 = hpdi(s, 0.5)
  print("PI50  =", np.round(PI50, 2))    # ≈[0.71, 0.93] 漏掉 p=1
  print("HPDI50=", np.round(HPDI50, 2))  # ≈[0.84, 1.00] 含峰，更窄
  ```
- **Worked example B — loss function 选点估计（书 R code 3.17–3.19）**：
  ```python
  import numpy as np
  from scipy.stats import binom
  p = np.linspace(0, 1, 1000); post = binom.pmf(3, 3, p); post /= post.sum()
  samples = np.random.default_rng(100).choice(p, size=20000, p=post)
  loss_abs = np.array([np.sum(post * np.abs(d - p)) for d in p])   # 绝对损失 |d-p|
  loss_sq  = np.array([np.sum(post * (d - p) ** 2) for d in p])    # 平方损失 (d-p)^2
  print("绝对损失最优 =", round(p[np.argmin(loss_abs)], 3), " ≈ 后验中位数", round(np.median(samples), 3))
  print("平方损失最优 =", round(p[np.argmin(loss_sq)], 3),  " ≈ 后验均值",   round(samples.mean(), 3))
  # 绝对损失→中位数(≈0.84)，平方损失→均值(≈0.80)：loss 不同，点估计不同
  ```
- **设计练习（OJ）**：实现 `hpdi(samples, prob)` 返回最窄区间 `(lo, hi)`。
  <details><summary>参考解与隐藏测试</summary>

  ```python
  # solution+tests: ch3 ex 3.2A
  import numpy as np
  from scipy.stats import binom
  def hpdi(samples, prob=0.5):
      x = np.sort(samples); n = len(x); w = int(np.floor(prob * n))
      widths = x[w:] - x[:n - w]; i = int(np.argmin(widths))
      return x[i], x[i + w]
  # 偏态后验 3/3：HPDI 必含峰(接近1)、且比 PI 窄
  p = np.linspace(0, 1, 1000); post = binom.pmf(3, 3, p); post /= post.sum()
  s = np.random.default_rng(1).choice(p, size=30000, p=post)
  lo, hi = hpdi(s, 0.5)
  assert hi > 0.97                                   # 含最可信值 p≈1
  assert (hi - lo) <= (np.percentile(s,75) - np.percentile(s,25)) + 1e-9  # 不比 PI 宽
  # 对称后验 6/9：HPDI ≈ PI
  post2 = binom.pmf(6, 9, p); post2 /= post2.sum()
  s2 = np.random.default_rng(2).choice(p, size=30000, p=post2)
  lo2, hi2 = hpdi(s2, 0.8); pi2 = np.percentile(s2, [10, 90])
  assert abs(lo2 - pi2[0]) < 0.05 and abs(hi2 - pi2[1]) < 0.05
  print("3.2A ok")
  ```
  </details>
- **关联书题**：3E4–3E7（分位数、66% PI/HPDI，p82）、3M2（90% HPDI，p82）。
- **常见误区**：
  - ❌ **"95% 可信区间 = 有 95% 概率含真值"当成频率派的意思** → 这是贝叶斯解读；频率派置信区间不能这么说（p71）。【对应错题本 active】
  - ❌ "偏态后验也用 PI" → PI 会漏掉最可信值，应看 HPDI 或干脆画整条后验（p70）。
  - ❌ "必须报一个点估计" → 整条后验才是估计，点估计常非必要；要报就先想清 loss function（p71–73）。

## 3.3 Sampling to simulate prediction（PDF p74–81）

- **学习目标**：从后验采样**模拟预测**，并用**后验预测检查（PPC）**评估模型。
- **核心概念**：
  - **dummy data 模拟数据**（§3.3.1, p74–76）：likelihood **双向**工作——给观测算可能性，给参数则定义一个**可采样的观测分布**。所以贝叶斯模型**天生是生成式的**（generative）。`np.random.binomial(n, p, size)` 模拟；`binom.pmf([0,1,2], 2, 0.7)=[.09,.42,.49]`。
  - **两种不确定性**（p78）：① **观测不确定**（即便 p 已知，下一次抛仍随机）② **参数不确定**（后验对 p 的不确定）。
  - **后验预测分布 posterior predictive distribution**（p78–79, Fig 3.6）：把参数不确定性**传播**进预测——对每个后验样本 p 各模拟一次观测再合并：`np.random.binomial(9, samples)`。**用整条后验而非点估计，预测才"诚实"**；只用峰值会**过度自信**。
  - **模型检查**（§3.3.2, p77–81）：retrodiction（重现训练数据）查软件；adequacy 查模型够不够用。**所有模型都是错的**，目标是看它**怎么错、错得要不要紧**。globe 例子：换个角度看数据——最长连续游程（observed=3）与水陆切换次数（observed=6）——切换次数暴露了"抛掷不独立"。
- **关键直觉**：**预测要把"对参数的不确定"一起带上**，否则会过度自信、误以为模型比实际更贴合数据（p79）。模型检查**主观但强大**：靠你的领域想象力换角度看数据，golem 想象力很差（p81）。
- **Worked example — 后验预测检查（书 R code 3.25–3.26）**：
  ```python
  import numpy as np
  from scipy.stats import binom
  p = np.linspace(0, 1, 1000); post = binom.pmf(6, 9, p); post /= post.sum()
  rng = np.random.default_rng(100)
  samples = rng.choice(p, size=10000, p=post)
  # 后验预测：对每个后验样本 p 模拟一次"9 抛中的水数"
  ppc = rng.binomial(9, samples)
  # 只用点估计(峰=0.667)的"过度自信"版做对比
  point = rng.binomial(9, 0.667, size=10000)
  print("后验预测 P(水数=6) =", round(np.mean(ppc == 6), 3))     # 观测 6 落在中央、最常见
  print("两者标准差:", round(ppc.std(), 2), "vs", round(point.std(), 2))  # 后验预测更宽(更诚实)
  ```
- **设计练习（OJ）**：实现 `posterior_predictive(samples, n, rng)` —— 对每个后验样本模拟 `n` 次伯努利的成功计数，返回长度同 `samples` 的预测计数数组。
  <details><summary>参考解与隐藏测试</summary>

  ```python
  # solution+tests: ch3 ex 3.3A
  import numpy as np
  from scipy.stats import binom
  def posterior_predictive(samples, n, rng):
      return rng.binomial(n, samples)
  p = np.linspace(0, 1, 1000); post = binom.pmf(6, 9, p); post /= post.sum()
  rng = np.random.default_rng(7)
  s = rng.choice(p, size=20000, p=post)
  ppc = posterior_predictive(s, 9, rng)
  assert ppc.shape == s.shape                       # 每个后验样本一条预测
  assert ppc.min() >= 0 and ppc.max() <= 9          # 计数在 0..9
  assert np.argmax(np.bincount(ppc, minlength=10)) in (6, 7)  # 众数≈观测6附近
  # 诚实性：后验预测比"只用点估计"更宽
  point = rng.binomial(9, 0.667, size=20000)
  assert ppc.std() > point.std()
  print("3.3A ok")
  ```
  </details>
- **关联书题**：3M3–3M4（后验预测检查，p82）、3H1–3H5（出生性别数据：后验、HPDI、PPC、独立性检查，p82–83）。
- **常见误区**：
  - ❌ "用后验峰值（点估计）做预测就行" → 丢掉参数不确定性 → **过度自信**（p79）。
  - ❌ "模型检查是要检验假设真不真" → 模型必错，目标是看**够不够用、怎么失败**（p78）。
  - ❌ "p 值式的尾部检查很强" → 它只按模型预期的角度看数据，是**弱检查**；换角度（游程/切换）更能为难模型（p81）。

## 3.4 Summary（PDF p81）
样本把"积分问题"变成"数据汇总问题"：可产区间、点估计、后验预测检查。**后验预测检查 = 参数不确定（后验）× 观测不确定（似然）** 的结合，用来验软件、找模型不足（p81）。
