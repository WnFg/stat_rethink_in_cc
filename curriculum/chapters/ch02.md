## 2.1 The garden of forking data（PDF p33–40，书页 20–27）

- **学习目标**：用"数花园里的路径"理解——概率推断就是"在所有可能里，数有多少条路径与观测一致"，再标准化成概率。
- **核心概念**（§2.1.1–2.1.3）：
  - **garden of forking data 分叉路径花园**（p34）：每个未知可能值是一条岔路；观测会"剪掉"与之不符的路径，剩下的就是逻辑上仍可能的。
  - **counting possibilities 数可能**（§2.1.1, p35–37）：袋里 4 颗弹珠、蓝/白未知 → 5 种 conjecture（猜想）。抽到序列 🔵⚪🔵，每种猜想能产生它的路径数 = 各层路径数相乘（p36）。
  - **用先验信息**（§2.1.2, p38）：有先验时，**新路径数 × 先验路径数**即可——这就是贝叶斯更新的雏形，且"全部重数"与"在旧计数上乘新计数"数学等价（前提：观测相互独立）。
  - **从计数到概率**（§2.1.3, p39–40）：计数只有相对大小有意义，且会爆炸增长，所以**标准化**为概率（除以总和）。并给出四个专有名词（p40）：参数 parameter（猜想的 `p`）、似然 likelihood（某个 p 产生数据的相对路径数）、先验概率 prior probability、后验概率 posterior probability。
- **关键直觉**：**"概率不是玄学，就是把'有多少种方式得到我看到的数据'标准化到 0–1"**（p39 honest ignorance 原则：能以更多方式产生数据的原因，更可信）。
- **Worked example**（教材 R code 2.1, p40 的 Python 等价：`ways/sum(ways)`）：
  ```python
  import numpy as np
  # 4 颗弹珠中蓝色颗数 ∈ {0,1,2,3,4}，对应 conjecture
  blue = np.array([0, 1, 2, 3, 4]); white = 4 - blue
  # 观测 🔵⚪🔵 -> 路径数 = blue * white * blue（各层相乘，p36）
  ways = blue * white * blue
  print(ways)            # [0 3 8 9 0]   <- 与书 p36 表一致
  print(ways / ways.sum())  # [0 0.15 0.40 0.45 0]  <- 书 R code 2.1
  ```
- **设计练习**（延伸 §2.1 的计数法）：
  1. **换数据数路径**：袋中 3 颗弹珠，蓝色颗数 ∈ {0,1,2,3}，观测序列 🔵🔵⚪。用 numpy 算每种猜想路径数（`blue*blue*white`）与标准化概率，指出最可信的猜想。（预期：ways=[0,2,4,0]→标准化[0,0.333,0.667,0]；蓝=2 唯一最可信。）
  2. **增量更新（呼应 §2.1.2, p38）**：接上原书弹珠例子，先验计数 `(0,3,8,9,0)`，又抽到一颗 🔵。把新路径数 `(0,1,2,3,4)` 乘上去得新计数，标准化后对比。（预期：新计数 `(0,3,16,27,0)`，与书 p38 表一致；"蓝多"的猜想更可信——体会"先验=上一轮后验"。）
- **关联书题**（counting method，p59–60）：**2M4**（三张卡 B/B,B/W,W/W，朝上是黑，另一面也是黑的概率=2/3，p59）、**2M5/2M6/2M7**（卡片计数进阶，p59–60）、**2M3**（Earth/Mars 地球仪，Pr(Earth|land)=0.23，p59）。均要求"用数路径法"。
- **常见误区**：
  - ❌ "似然必须自己加起来=1" → 似然是相对路径数，未必归一；标准化后才是概率（p39）。
  - ❌ "贝叶斯更新要重算一切" → 旧计数×新计数即可，独立时与全部重数等价（p38）。

## 2.2 Building a model（PDF p41–44，书页 28–31）

- **学习目标**：把真实问题"地球表面水比例 `p`"写成贝叶斯模型，并直观看它如何从数据学习。
- **核心概念**：
  - **globe tossing 例子**（p41）：抛接地球仪，记右手食指下是水(W)/陆(L)。观测 `W L W W W L W L W` → 6 W、3 L、N=9。要估计参数 `p`。
  - **三步建模循环**（p41）：① **data story 数据故事**（叙述数据怎么来的）② **update 更新**（喂数据）③ **evaluate 评估**（检查、可能修模型）。
  - **data story**（§2.2.1, p42）：每次抛接独立、W 概率恒为 p、1−p 出 L —— 这正对应二项过程。
  - **Bayesian updating 贝叶斯更新**（§2.2.2, p42–44, Fig 2.5）：从先验（均匀直线）开始，每来一个观测就把可信度曲线更新一次；见到 W 峰右移、见到 L 峰左移，数据越多曲线越窄越高。**更新可前可后、与数据顺序无关**（p44）。
- **关键直觉**：模型每次只把"上一轮后验"当"这一轮先验"，逐点学习（p44）。**学生注意（直接回应你的硬币直觉）**：§2.2 的 Rethinking「Sample size and reliable inference」(p44) 明确说——**贝叶斯估计对任何样本量都有效**，小样本一样有正当解释，代价是依赖先验。你说"样本太少、靠先验、约 1/2"正是这个道理。
- **Worked example**（用 §2.1 的网格逻辑模拟 Fig 2.5 的逐步更新，纯 numpy；正式网格法在 §2.4）：
  ```python
  import numpy as np
  from scipy.stats import binom
  data = "WLWWWLWLW"
  p = np.linspace(0, 1, 100)
  post = np.ones_like(p)                 # 均匀先验
  for i, obs in enumerate(data, 1):
      w = 1 if obs == "W" else 0
      post = post * binom.pmf(w, 1, p)   # 每次一个观测做更新
      post = post / post.sum()
      # 可在此画出第 i 步的曲线，对应 Fig 2.5
  print("峰值 p≈", round(p[np.argmax(post)], 3))  # ≈0.667
  ```
- **设计练习**：
  1. 用自己的话写 globe tossing 的"数据故事"，并说明为什么对应 Binomial(N, p)（呼应 p42 的三条假设）。
  2. **顺序无关性（验证 p44）**：把 `data` 打乱顺序（仍 6W3L）重跑上面循环，确认最终后验完全相同。说说这对应书中哪条性质。（预期：最终曲线不变；对应"数据可任意顺序/一次性喂入"。）
- **关联书题**：**2H1–2H4**（熊猫双胞胎，p60，完整的多步贝叶斯更新+不同类型数据，作本章 Hard 挑战）。
- **常见误区**：
  - ❌ "p 是数据" → p 是未知**参数**，N 和 w 才是数据（p47）。
  - ❌ "模型很自信 = 模型很对" → 自信只是"在这个模型设定下"，模型可能严重错误仍很自信（p43 evaluate 警告）。

## 2.3 Components of the model（PDF p45–50，书页 32–37）

- **学习目标**：拆解贝叶斯模型四零件——likelihood / parameter / prior / posterior，并理解 Bayes 定理如何把它们拼起来。
- **核心概念**（§2.3.1–2.3.4）：
  - **likelihood 似然**（§2.3.1, p45–46）：由数据故事推出 `w ~ Binomial(n, p)`，即 `Pr(w|n,p)=C(n,w) p^w (1−p)^{n−w}`。书 R code 2.2：`dbinom(6,9,0.5)=0.164`。
  - **parameters 参数**（§2.3.2, p47）：似然里可调的量。globe tossing 中 n、w 是数据，`p` 是要估的参数。
  - **prior 先验**（§2.3.3, p47–49）：参数的初始可信度。这里用 `p ~ Uniform(0,1)`（p48）。书强调先验是"工程假设"，**regularizing / weakly informative prior（正则化/弱信息先验）**通常更好（p48），且应像其他假设一样被检验、改动。
  - **posterior 后验**（§2.3.4, p49–50）：`Pr(p|n,w)`，由 **Bayes 定理**给出。
- **关键直觉**：**Posterior = (Likelihood × Prior) / Average Likelihood**（p50 文字式）。分母 `Pr(w)=∫Pr(w|p)Pr(p)dp` 只是把后验标准化到积分为 1（p50）。核心一句：**"后验 ∝ 先验 × 似然"，因为路径数 = 先验路径数 × 新路径数，乘法只是压缩了的计数**（p50）。
- **Worked example**（书 R code 2.2, p46 的 Python 等价）：
  ```python
  from scipy.stats import binom
  print(binom.pmf(6, 9, 0.5))   # 0.1640625，与书 p46 一致：给定 p=0.5 看到 6/9 水的相对路径数
  ```
- **设计练习**：
  1. **似然与 MLE**：固定 W=6,N=9，对 `p∈{0,.25,.5,.75,1}` 算 `binom.pmf(6,9,p)`，找似然最大的 p。（预期：p=6/9≈0.667 最大；这正是频率派的 MLE——铺垫 §2.4.2。）
  2. **先验的作用（呼应 Fig 2.6, p51）**：把先验从 Uniform 换成"step 先验"（p<0.5 取 0，否则取常数），定性说后验怎么变。（预期：p<0.5 被清零、后验截断右移。）
- **关联书题**：**2E1–2E4**（概率表达式的含义与条件概率，p58–59；2E4 借 de Finetti「PROBABILITY DOES NOT EXIST」讨论"水概率=0.7"到底指什么——很适合澄清贝叶斯概率观）。
- **常见误区**：
  - ❌ "贝叶斯就是加了个先验的玄学 / posterior 就是 prior" → 后验是先验与似然之积再标准化；数据多时似然主导（p50, p46 Rethinking）。
  - ❌ 混淆 `Pr(w|p)`（似然）与 `Pr(p|w)`（后验）（p46）。

## 2.4 Making the model go（PDF p50–57，书页 37–44）

- **学习目标**：掌握把后验算出来的三种引擎（p52）：**grid / quadratic / MCMC**，重点动手网格近似，并直觉理解另两者。
- **核心概念**（§2.4.1–2.4.3）：
  - **Grid approximation 网格近似**（§2.4.1, p53）：把连续的 p 切成有限格点，每格 `prior×likelihood`，**除以总和标准化**。书 R code 2.3 用 20 个点。简单、直观，但**随参数增多急剧变慢**（p54：2 参数×100 点=1e4，10 参数=数十亿），故主要作教学工具。
  - **Quadratic approximation 二次近似**（§2.4.2, p54–57）：后验峰附近近似高斯，用 **MAP（maximum a posteriori，后验峰）+ 曲率（Hessian，二阶导）** 两个数刻画。书用 `map()` 得 `p: Mean 0.67, StdDev 0.16, 89% 区间 [0.42, 0.92]`（p55）。数据越多越准（Fig 2.8, n=9/18/36, p56）。**uniform 先验或大样本下，quap ≈ MLE + 标准误**（p57 Rethinking）。
  - **MCMC**（§2.4.3, p57）：不直接算后验，而是**从后验抽样**，用样本频率重建后验。复杂/多层模型的主力，第 8 章详讲。
- **关键直觉**：三种引擎都在逼近**同一个**后验，只是算法不同；网格法最适合理解后验的形状（p53）。
- **Worked example A — 网格近似（书 R code 2.3, p53 的 Python 等价，sum 归一化）**：
  ```python
  import numpy as np
  from scipy.stats import binom

  def grid_posterior(W, N, n_grid=20, prior=None):
      p = np.linspace(0, 1, n_grid)
      if prior is None:
          prior = np.ones_like(p)                 # Uniform(0,1)
      likelihood = binom.pmf(W, N, p)
      unstd = likelihood * prior
      posterior = unstd / unstd.sum()             # 书用 sum 归一化（网格点上的概率质量）
      return p, posterior

  p, post = grid_posterior(6, 9, 20)
  print("后验均值≈", round(np.sum(p * post), 3))      # ≈0.636
  print("峰值(MAP)≈", round(p[np.argmax(post)], 3))   # 20 点→0.684；1000 点→0.667
  ```
- **Worked example B — 二次近似（书 §2.4.2 `map` 的 Python 等价：optimize 找 MAP + 数值二阶导）**：
  ```python
  import numpy as np
  from scipy.stats import binom, norm
  from scipy.optimize import minimize

  neg_log_post = lambda p: -binom.logpmf(6, 9, np.clip(p, 1e-9, 1-1e-9))  # uniform 先验下 ∝ -loglik
  res = minimize(neg_log_post, 0.5, method="L-BFGS-B", bounds=[(1e-6, 1-1e-6)])
  map_p = res.x[0]
  eps = 1e-4                                   # 数值二阶导 ≈ Hessian
  second = (neg_log_post(map_p+eps) - 2*neg_log_post(map_p) + neg_log_post(map_p-eps)) / eps**2
  sd = 1/np.sqrt(second)
  print(f"MAP={map_p:.2f}, SD={sd:.2f}")                       # 0.67, 0.16 —— 与书 p55 一致
  print("89% 区间:", np.round(norm.ppf([0.055, 0.945], map_p, sd), 2))  # [0.42 0.92]
  ```
- **设计练习**：
  1. **网格密度（书 p53 建议）**：用 example A 画 W=6,N=9 后验曲线（matplotlib，存 `materials/figures/`），把 `n_grid` 取 5/20/100/1000 对比。（预期：5 点很粗糙，20 点已不错，>100 后基本不变——复现 Fig 2.7。）
  2. **两种先验（书 R code 2.5, p53）**：分别用 step 先验 `(p>=0.5).astype(float)` 和峰形先验 `np.exp(-5*np.abs(p-0.5))` 重算并对比均匀先验。（预期：复现 Fig 2.6——step 截断、peaked 收窄并偏移。）
  3. **quap vs 网格 vs 解析（书 Fig 2.8, p56）**：对 (W,N)=(6,9)、(12,18)、(24,36)，把 example B 的高斯近似与 example A 的网格后验、解析后验 `scipy.stats.beta(W+1, N-W+1)` 叠在一张图比。（预期：数据越多三者越接近，n=9 时高斯在右侧偏差明显、甚至给 p=1 正密度——这是 quap 的缺陷。）
- **关联书题**：**2M1**（对 W,W,W / W,W,W,L / L,W,W,L,W,W,W 各做网格近似，均匀先验，p59）、**2M2**（同上但用 step 先验，p59）。
- **常见误区**：
  - ❌ "网格近似是精确解" → 它是离散逼近，格点越密越准但永远近似（p53）。
  - ❌ 忘记标准化就拿 `likelihood*prior` 当后验去比较（p53 第 5 步）。
  - ❌ "二次近似总成立" → 仅当后验近高斯（常见于大样本）才好；小样本/偏态会失真（p56–57, Fig 2.8）。
  - ❌ 把 quap 的 MAP 当成"唯一正确答案" → 它在 uniform 先验下≈MLE（p57），但贝叶斯关心的是**整条后验分布**，不是一个点。

## 2.5 Summary（PDF p57）
贝叶斯推断的目标是**后验分布**；模型 = 似然 + 参数 + 先验，由 Bayes 定理得后验；实务中用 grid / quadratic / MCMC 三种数值引擎拟合，各有取舍（p57）。
