# 第 2 章 Small Worlds and Large Worlds — 教案

**本章定位**：用最简单的"globe tossing（抛地球仪）"例子，把贝叶斯推断的整套零件讲透——likelihood、parameter、prior、posterior，以及让模型"动起来"的三种引擎（grid / quadratic / MCMC）。这是全书的地基。
**里程碑**：能用网格近似（grid approximation）手写出 globe tossing 的 posterior 并解读它。
**数据**：无需外部数据，例子里的观测序列 `W L W W W L W L W`（9 抛 6 水）直接构造。
**工具**：NumPy（+ SciPy 在 2.4 二次近似）。本章不需要 PyMC。

## 2.1 The garden of forking data（PDF p33–40）

- **学习目标**：用"数路径"（counting paths）理解：概率推断本质是"在所有可能里，数有多少种走法和数据一致"。
- **核心概念**：
  - garden of forking data（分叉路径花园）：每个未知的可能取值是一条岔路；观测数据会"砍掉"与之不符的路径。
  - 似然 likelihood = 给定某个假设，能产生观测数据的路径条数（相对大小）。
  - 贝叶斯更新 Bayesian updating = 新数据到来时，把路径计数累乘。
- **关键直觉**：袋子里有 4 颗弹珠，蓝/白未知（5 种构成：⚪⚪⚪⚪ … 🔵🔵🔵🔵）。抽到"🔵⚪🔵"这一序列，每种构成能产生它的路径数不同——路径数最多的构成最 plausible。**概率不是玄学，就是把"有多少种方式得到我看到的数据"标准化成 0–1。**
- **Worked example**（数路径 → 归一化为 plausibility）：
  ```python
  import numpy as np
  # 4 颗弹珠中蓝色的颗数 p_blue ∈ {0,1,2,3,4}
  blue = np.array([0, 1, 2, 3, 4])
  white = 4 - blue
  # 观测序列 🔵⚪🔵：第1抽蓝、第2抽白、第3抽蓝 -> 路径数 = blue * white * blue
  ways = blue * white * blue
  plausibility = ways / ways.sum()
  print(ways)          # [0 3 8 9 0]
  print(plausibility)  # 归一化后的相对可信度
  ```
- **设计练习**：
  1. **复用计数法**：袋子改为 3 颗弹珠，蓝色颗数 ∈ {0,1,2,3}。观测序列为"🔵🔵⚪"。用 numpy 算每种构成的路径数与 plausibility，并说出哪种最可信。（预期结论：ways=[0,2·... ]；蓝=2 时路径数最大。让学生自己推 `blue*blue*white`。）
  2. **增量更新**：接着上一题，又抽到一颗 🔵（放回）。把新观测的路径数乘到旧 ways 上，看 plausibility 怎么变。（预期结论：新数据让"蓝多"的构成更可信——体会"先验=上一轮后验"。）
- **关联书题**：2M7（路径计数进阶，H 偏概念）。
- **常见误区**：
  - ❌ "似然就是概率，必须自己加起来等于 1" → 似然是相对路径数，未必归一；归一化后才是 plausibility/概率。
  - ❌ "贝叶斯更新需要重算一切" → 旧后验可当新先验，乘上去即可（增量）。

## 2.2 Building a model（PDF p41–44）

- **学习目标**：把一个真实问题（地球表面有多少比例是水 `p`）写成一个贝叶斯模型的三要素。
- **核心概念**：globe tossing 实验——抛接地球仪，记右手食指下是水(W)还是陆(L)。观测 `W L W W W L W L W`（N=9，水 W=6）。要估计参数 `p`=水的比例。
- **关键直觉**：从"数弹珠路径"无缝过渡到"连续参数 `p`"。p 可以是 0–1 间任意值，每个 p 都是花园里的一条岔路。
- **Worked example**：写出数据故事（data story）：每次抛接独立、水的概率为 p → 这正是二项过程。
  ```python
  # 观测
  W, L = 6, 3   # 6 水 3 陆，共 9 次
  ```
- **设计练习**：
  1. 用自己的话写出 globe tossing 的"数据故事"，并说明为什么它对应二项分布 Binomial(N, p)。（预期结论：独立、二值、同一 p → 二项。）
- **关联书题**：—
- **常见误区**：❌ "p 是数据" → p 是未知**参数**，数据是 W/L 的序列。

## 2.3 Components of the model（PDF p45–49）

- **学习目标**：分清贝叶斯模型的四个零件：likelihood、parameter、prior、posterior，以及它们怎么由贝叶斯定理拼起来。
- **核心概念**：
  - **likelihood 似然**：`W ~ Binomial(N, p)`，即 `P(W | p) = C(N,W) p^W (1-p)^(N-W)`。
  - **parameter 参数**：`p`，我们要推断的未知量。
  - **prior 先验**：`p ~ Uniform(0,1)`（一开始一无所知）。
  - **posterior 后验**：`P(p | W) ∝ P(W | p) · P(p)`。
- **关键直觉**：**posterior ∝ prior × likelihood**。这一行是全书最重要的等式——先验是出发点，似然是数据的"投票"，后验是两者相乘再归一化。
- **Worked example**：
  ```python
  import numpy as np
  from scipy.stats import binom
  p = 0.5
  print(binom.pmf(6, 9, p))   # 给定 p=0.5，看到 6/9 水的似然
  ```
- **设计练习**：
  1. 固定数据 W=6,N=9，对 `p` 取 0,0.25,0.5,0.75,1，分别算 likelihood `binom.pmf(6,9,p)`。哪个 p 的似然最大？它等于多少？（预期结论：p=6/9≈0.667 似然最大，即 MLE。）
  2. 概念题：把"先验 Uniform(0,1)"换成"先验只允许 p≥0.5"，定性说后验会怎么变。（预期结论：p<0.5 区域被先验清零，后验整体右移/截断。）
- **关联书题**：2E1–2E4（区分各种概率表述，E 概念题）。
- **常见误区**：
  - ❌ "posterior 就是 prior" / "贝叶斯只是加了个先验" → 后验是 prior 与 likelihood 的乘积归一，数据多时似然主导。
  - ❌ 把 `P(W|p)` 和 `P(p|W)` 当成一回事（似然 ≠ 后验）。

## 2.4 Making the model go（PDF p50–57）

- **学习目标**：掌握把后验"算出来"的三种引擎，重点动手 **grid approximation（网格近似）**，并直觉理解 quadratic approximation 与 MCMC。
- **核心概念**：
  - **Grid approximation 网格近似**：把 p 切成有限格点，每格算 prior×likelihood，归一化 → 近似后验。简单、直观，维度高时不可行。
  - **Quadratic approximation 二次近似**（又名 Laplace/正态近似）：用一个正态分布去逼近后验峰附近，靠找 MAP（峰值）+ 曲率（Hessian）。
  - **MCMC**：用随机游走从后验采样，后面第 8 章详讲。
- **关键直觉**：三种方法都是为了得到/逼近同一个后验，只是"算法引擎"不同；网格法是理解后验形状的最佳教学工具。
- **Worked example**（网格近似 globe tossing 后验）：
  ```python
  import numpy as np
  from scipy.stats import binom

  def grid_posterior(W, N, n_grid=200, prior=None):
      p_grid = np.linspace(0, 1, n_grid)
      if prior is None:
          prior = np.ones_like(p_grid)          # Uniform(0,1)
      likelihood = binom.pmf(W, N, p_grid)
      unstd = likelihood * prior
      posterior = unstd / np.trapezoid(unstd, p_grid)  # 归一化为密度
      return p_grid, posterior

  p_grid, post = grid_posterior(6, 9)
  # 后验均值与峰值
  mean = np.trapezoid(p_grid * post, p_grid)
  mode = p_grid[np.argmax(post)]
  print(f"后验均值≈{mean:.3f}, 峰值(MAP)≈{mode:.3f}")  # 约 0.636 / 0.667
  ```
- **设计练习**：
  1. **动手网格法**：用上面 `grid_posterior` 画出 W=6,N=9 的后验曲线（matplotlib），并标出峰值。再把 `n_grid` 从 5 改到 1000，观察网格点太少时后验有多粗糙。（预期结论：网格越密越平滑；峰值稳定在≈0.667。）
  2. **先验的影响**：用 step 先验 `prior = (p_grid >= 0.5).astype(float)`（只信 p≥0.5）重算后验，和均匀先验对比作图。（预期结论：p<0.5 部分被压成 0，后验右移，峰值不变但形状变陡。）
  3. **数据的影响（复用 2.3）**：分别用 (W,N)=(3,3)、(6,9)、(12,18) 跑网格法。三者 W/N 同为 2/3，但 N 越大后验越窄。作图说明"数据越多越确定"。（预期结论：后验随 N 增大而集中收窄。）
- **关联书题**：2M1–2M3（网格近似不同数据/先验，M 动手）、2H1–2H4（综合应用，H 挑战）。
- **常见误区**：
  - ❌ "网格近似是精确解" → 它是离散逼近，格点越密越准但永远近似。
  - ❌ 忘记归一化就把 `likelihood*prior` 当后验密度去比较/积分。
  - ❌ "二次近似总对" → 仅当后验近似正态（常见于样本量大）时好用，偏态后验会失真。
